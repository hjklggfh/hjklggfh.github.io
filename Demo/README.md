# 纯前端 Canvas 像素风修仙开放世界 RPG

当前阶段聚焦地图生成系统，同时保留后续扩展修仙玩法的工程结构。

## 技术栈

- TypeScript
- HTML5 Canvas2D
- 原生 Web API
- requestAnimationFrame
- Vite 仅作为本地开发和打包工具

没有使用 Unity、Unreal、PixiJS、Phaser、Three.js。

## 运行

```bash
npm install
npm run dev
```

浏览器打开 Vite 输出的本地地址。

## 操作

- `WASD` / 方向键：移动
- `E` / 空格：交互
- `I`：背包
- `C`：角色面板
- `F5` 或 HUD 存档按钮：保存
- 鼠标滚轮：缩放
- 移动端：左侧虚拟摇杆，右侧交互键

## 素材自动识别

工程会扫描以下目录中的图片：

- `src/assets`
- `assets`
- `game-assets`
- `tilesets`
- `tiles`
- `sprites`
- `resources`
- `art`
- `素材`

支持格式：`png`、`jpg`、`jpeg`、`webp`、`gif`、`bmp`。

识别逻辑会根据文件名关键词分类，并自动：

- 推断 tile 大小，优先适配 `32×32`
- 对整张 tileset 自动切片
- 为每个切片建立 tile 索引
- 将识别到的素材映射到世界使用的草地、水体、道路、桥梁、森林、山脉、房屋、围栏、灯笼等 Tile
- 没有素材时使用程序化像素后备图块

浏览器支持 File System Access API 时，也可点击 HUD 的“素材”按钮选择本地素材目录并热加载。

## 已实现系统

- 500×500 Tile 世界地图
- 32×32 Tile
- 32×32 Tile Chunk 流式渲染
- 摄像机裁剪，仅绘制可见 Chunk
- 确定性世界种子
- 低频噪声地形生成
- 自然河流、宽度变化、曲线路径、桥梁
- 湖泊、岸线
- 连续山脉和阻挡路径
- 密度变化森林、自然边缘、林间道路、空地
- 主道路、支路、隐藏区域、自然边界
- 村庄、小镇、独立房屋、修仙坊市
- 自动房屋、道路、围栏、灯笼、树木布局
- 村庄围栏自动开门，门口铺路并保持可通行
- 村庄内部最多保留一个装饰性池塘，避免水体阻断生活区
- 更紧凑的房屋占地，带屋顶、墙体、门、窗和屋檐图块
- 昼夜循环、天气、雾、雨、水面动画、草地摆动
- 玩家移动、碰撞、摄像机跟随、探索遮罩
- NPC 随机刷新、自由走动、靠近交互
- NPC 对话树、好感度、交易、赠送、切磋、战斗入口
- 背包分类筛选
- 角色面板、属性、装备槽、功法列表
- localStorage 本地存档

## 目录结构

```text
src/
  assets/          内置程序化 tile 与用户素材入口
  core/            游戏主循环、资源、摄像机、随机、噪声、数学工具
  entities/        Player、NPC、实体接口
  input/           键盘、鼠标、移动端摇杆输入
  inventory/       物品与背包模型
  render/          Canvas2D Chunk 渲染器
  save/            本地存档
  ui/              HUD、面板、对话
  world/           世界常量、Tile 类型、地图数据、生成器、氛围系统
```
